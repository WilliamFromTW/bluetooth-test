# 提案：歷史日誌自動儲存與管理功能

## 背景與問題描述
目前系統的日誌僅保存在前端記憶體中，當網頁重新整理或關閉後，所有除錯日誌就會消失。為了方便後續問題追蹤與分析，我們需要一套能自動儲存並管理歷史日誌的機制。

## 解決方案
我們將實作「後端儲存 + 前端自動發送」的架構：
1. **前端 UI**：在「設定與日誌」分頁中加入「自動保存日誌」開關，以及「歷史日誌管理」按鈕。
2. **後端 API**：新增 `/api/logs` 相關路由，接收前端傳送的日誌字串，並以日期為檔名（如 `2026-06-23.log`）附加寫入伺服器本機檔案。
3. **歷史日誌 Modal**：允許使用者按日期篩選日誌、匯出日誌內容，或清空特定日期的紀錄。

## 影響範圍
- 前端：`index.html`, `app.js`, `style.css`, `i18n.js`
- 後端：`main.py`
