## ADDED Requirements

### Requirement: 後端日誌儲存
系統必須（MUST）將前端發送的日誌，以日期為檔名（格式為 `YYYY-MM-DD.log`）儲存於伺服器本機的 `logs/` 資料夾中。

#### Scenario: 成功儲存當日日誌
- **WHEN** 前端啟用自動保存，並發送了一則新的日誌訊息給 `/api/logs`
- **THEN** 後端必須將該訊息附加（append）寫入伺服器本機的 `logs/YYYY-MM-DD.log` 檔案中，並回傳成功狀態

### Requirement: 歷史日誌讀取與清單
系統必須（MUST）提供 API 供前端取得所有可用的歷史日誌日期清單，以及特定日期的詳細內容。

#### Scenario: 讀取可用日期清單
- **WHEN** 前端呼叫 `GET /api/logs/dates`
- **THEN** 系統必須掃描 `logs/` 資料夾，並以陣列形式回傳可用的日期字串清單（例如 `["2026-06-23", "2026-06-22"]`）

#### Scenario: 讀取特定日期內容
- **WHEN** 前端呼叫 `GET /api/logs/{date}` 並提供有效日期
- **THEN** 系統必須回傳該日期對應日誌檔的純文字內容

### Requirement: 歷史日誌刪除
系統必須（MUST）允許使用者永久刪除特定日期的歷史日誌紀錄。

#### Scenario: 成功刪除紀錄
- **WHEN** 使用者在前端確認要刪除某日的紀錄，並呼叫 `DELETE /api/logs/{date}`
- **THEN** 系統必須將該日期對應的日誌檔案從 `logs/` 目錄中移除，並回傳成功狀態

### Requirement: 前端歷史日誌介面與多語系
系統必須（MUST）在前端提供歷史日誌管理介面，包含開啟彈窗的按鈕、日期選擇下拉選單、內容檢視區塊以及匯出/刪除按鈕，且所有文字皆需支援多國語系。

#### Scenario: 開啟歷史日誌彈窗
- **WHEN** 使用者點擊「歷史日誌」按鈕
- **THEN** 系統必須顯示檢視器彈窗，並自動拉取可用的日期清單填充下拉選單

#### Scenario: 匯出歷史紀錄
- **WHEN** 使用者點擊「匯出為 TXT」按鈕
- **THEN** 系統必須將畫面上的日誌內容轉換為文字檔，並觸發瀏覽器下載（檔名包含日期）
