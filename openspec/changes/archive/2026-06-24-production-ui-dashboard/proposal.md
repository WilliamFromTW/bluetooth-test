## Why

目前系統採用多頁籤（Tabs）設計，需要來回切換才能進行連線、發送指令與查看日誌等操作。這對於在生產線環境的作業員來說操作不夠直覺，容易分散注意力並造成誤操作。為了提升產線測試效率，我們需要一個無需捲動視窗（固定高度無捲軸）、大字體、高防呆的緊湊單頁佈局。

## What Changes

- 取消頁籤設計，將所有必要的測試資訊整合在同一頁面上（左側設備選單、右側測試操作與結果）。
- 將現有 UUID 等工程師專用設定移至「進階設定」的隱藏式彈窗（Modal）中。
- 將連線狀態與 MAC address 整合為一顆明顯的狀態卡片按鈕，點擊可直接觸發斷開連線，捨棄原本獨立的斷開連線按鈕。
- 將互動日誌訊息微縮，主畫面僅顯示最新的一條狀態，並將詳細內容移入歷史日誌彈窗中。
- 鎖定容器最大高度 (`100vh`)，徹底消除頁面的垂直捲軸，確保固定版面。

## Capabilities

### New Capabilities
- `production-ui`: 提供給生產線人員操作的極簡、單頁與防呆化主控面板與佈局機制。

### Modified Capabilities
- `ble-web-dashboard`: 調整現有主控台介面排版與前端互動邏輯以符合單頁設計，修改元件佈局與互動綁定。

## Impact

- `static/index.html`: DOM 結構將進行大幅重構，拔除 Tabs 架構。
- `static/style.css`: 新增單頁網格佈局、卡片式設計與設定彈窗的專屬樣式，確保 100vh 鎖定。
- `static/app.js`: 配合新的 DOM ID 更新事件監聽器（例如處理整合狀態按鈕的點擊），以及「設定 Modal」的控制邏輯。
