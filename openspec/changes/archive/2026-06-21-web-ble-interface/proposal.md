## Why

為了解決藍牙產品測試與開發過程中，缺乏直觀且易於使用的圖形化介面（GUI）問題。目前缺乏直觀介面，這使得進行藍牙連線、UUID 設定、讀寫特徵值以及訂閱通知等操作時不夠便利。建立一個 Web 介面能讓使用者更快速地與藍牙產品進行互動、發送命令並即時查看回應，從而提升研發與測試效率。同時，目前的工具常常會列出一堆未上電或快取的設備（例如以前使用 Visual Studio UWP 時遇到的問題），造成混淆，需要一個能真實過濾並列出實際可用設備的介面。

## What Changes

- 新增一個響應式 Web 網頁前端介面，讓使用者可以：
  - **列出藍牙設備**：無須事先確認是否符合 Service UUID，也無須配對，即可列出周遭的藍牙設備。
  - **過濾功能**：可透過設備名稱（Name）或 MAC Address 進行過濾，且過濾設定會被儲存起來，以便日後修改與繼續使用。
  - **真實設備過濾**：特別處理排除未上電或系統快取的設備，確保列出的都是真實且開啟藍牙的活躍設備。
  - **連線與設定**：從列表中選擇設備並建立連線，連上指定的 Service UUID。
  - **Notify 設定**：連線後，啟用 Read UUID 的 Notify 功能以接收資料。
  - **發送指令**：提供指令輸入區發送資料至 Write UUID。支援切換「Binary」（例如 `FF FF AF`）或「Text」模式。
  - **字元觸發自訂函式**：在指令區輸入時，每個字元的輸入都能觸發特定的 Function，方便後續客製化該段程式邏輯。
- 新增一個基於 Python (FastAPI + bleak) 的後端服務，負責處理底層的藍牙掃描、連線、過濾快取設備，以及與藍牙特徵值（Characteristic）的讀寫與通知（Notify）互動。
- 提供即時連線（如 WebSocket），確保藍牙裝置接收到的掃描結果與通知能即時推送並顯示在 Web 前端介面上。

## Capabilities

### New Capabilities
- `ble-device-discovery`: 提供真實且活躍的藍牙設備掃描機制，過濾無效或快取的設備紀錄。
- `ble-web-dashboard`: 提供使用者過濾設備、選擇連線、設定 UUID、切換 Binary/Text 模式發送命令、訂閱通知、以及每個字元觸發客製化程式的網頁介面。
- `ble-backend-service`: 提供藍牙掃描、連線、讀寫特徵值、以及訂閱通知的後端 API（包含 REST 與 WebSocket）。

### Modified Capabilities
(無)

## Impact

- **API 接口**：新增後端 API 路由與 WebSocket 路由，用於藍牙設備掃描結果傳遞及即時資料流傳輸。
- **儲存機制**：前端需實作機制（如 LocalStorage）儲存使用者的過濾設定。
- **依賴項目**：後端新增 `fastapi`、`uvicorn`、`websockets` 等 Python 套件。需特別注意作業系統藍牙快取處理機制的實作。
- **硬體與作業系統**：需要執行該程式的系統具備藍牙硬體支援，且作業系統權限允許 Python 存取藍牙模組。
