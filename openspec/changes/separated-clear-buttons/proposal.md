## Why

在目前的「測試結果」版面中，回應日誌與測試判定結果（NG/OK）同時顯示在左右兩側。產線人員在反覆測試同一台設備時，會需要獨立保留其中一邊的資訊（例如只重置判定結果，或是只清空舊的日誌來送新指令），因此需要將兩者的清除功能分離。

## What Changes

- 在測試結果區的標題列最右側，新增兩個獨立的按鈕：`[清除紀錄]` 與 `[清除結果]`。
- 點擊 `[清除紀錄]` 時，僅會清空左側的通訊回應資料。
- 點擊 `[清除結果]` 時，僅會清空右側的判定結果（NG/OK 字樣）。
- 左右兩側的測試結果顯示框內，不得放置任何控制按鈕或其他雜訊，維持最乾淨的顯示。

### New Capabilities

### Modified Capabilities
- `production-ui`: 修改主控台畫面的行為，增加測試結果區域的獨立清除按鈕控制功能。

## Impact

- 影響前端 `static/index.html` 的 UI 結構，需要在原測試結果標題處加入兩個按鈕。
- 影響前端 `static/app.js`，需要綁定這兩個新按鈕的 Click 事件。
- 由於只是前端 UI 的微調，不影響後端 API 與藍牙通訊核心。
