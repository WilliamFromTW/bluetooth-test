## Context

目前的「測試結果」版面設計中，為了因應產線大字報需求，移除了所有多餘的視覺元素。但發現測試過程中，前一次的回應紀錄或判定結果（NG/OK）會殘留在畫面上，導致混淆。需要建立簡單、分離的清除機制。

## Goals / Non-Goals

**Goals:**
- 在不破壞測試結果大字報版面潔淨度的前提下，新增「清除紀錄」與「清除結果」按鈕。
- 確保按鈕僅清除對應區域的資料。

**Non-Goals:**
- 不更動後端 API 與掃描邏輯。
- 不改變原本的日誌檔案儲存邏輯，這裡的「清除紀錄」僅指畫面上的顯示清除。

## Decisions

- **UI 放置位置**：將兩個清除按鈕置於整個「測試結果」區塊的標題列最右側。理由是這是唯一的非資料顯示區域，放置控制按鈕最不突兀。
- **功能實作方式**：在 `static/app.js` 綁定 click 事件，點擊時分別對左側的回應資料區塊與右側的狀態區塊設定 `innerHTML = ''`。

## Risks / Trade-offs

- [Risk] 無。純粹的前端 DOM 操作，不涉及複雜狀態管理。
